# DataAligner: Production-Ready Multimodal Crypto Sentiment Dataset

## Overview

`DataAligner` is a production-grade Python class that orchestrates the alignment of 8 heterogeneous data sources into a single, high-quality dataset for machine learning models. Designed specifically for cryptocurrency market sentiment forecasting, it handles:

✅ **Market Data**: OHLCV (klines), funding rates  
✅ **On-Chain Metrics**: Open interest, liquidations, long/short ratios  
✅ **Sentiment Indices**: Fear/Greed index, macroeconomic news sentiment  
✅ **Text Data**: Aggregated Reddit discussions and crypto news  
✅ **Continuous Targets**: Volatility-adjusted sentiment regression scores (-100 to +100)  
✅ **Multimodal Features**: Chart images embedded in Hugging Face Dataset  
✅ **Data Integrity**: Strict forward-fill merging (no data leakage), automatic image validation

**Result:** Production-ready Dataset ready for deep learning (transformers, vision models, multimodal fusion networks)

## Class API Reference

### Initialization

```python
DataAligner(
    asset: str = "BTC",           # "BTC" or "ETH"
    data_dir: str = "data",       # Root directory (contains raw/ and processed/)
    image_dir: str = "data/processed/images",  # Chart images directory
    horizon_hours: int = 24       # Prediction horizon for target
)
```

**Parameters:**
- `asset`: Cryptocurrency symbol ("BTC" or "ETH"). Normalized to uppercase.
- `data_dir`: Root data directory. Must contain `raw/` subdirectory with CSV files.
- `image_dir`: Path to processed candlestick chart images. Generated by `ChartGenerator`.
- `horizon_hours`: Hours ahead for computing sentiment targets. Default 24 hours.

**Raises:**
- `ValueError`: If asset is not "BTC" or "ETH"
- `FileNotFoundError`: If raw data directory doesn't exist

### Main Methods

#### `load_all_data() → None`

Loads and merges all 8 data sources:

**Hourly Datasets (Exact Join):**
1. `coinalyze_open_interest.csv` → `open_interest_open/high/low/close`
2. `coinalyze_liquidations.csv` → `long_liquidations`, `short_liquidations` (filled 0)
3. `coinalyze_long_short_ratio.csv` → `long_short_ratio`, `long_percentage`, `short_percentage`
4. `gdelt_macro.csv.csv` → `news_volume`, `avg_sentiment_tone` (filled 0)

**Lower-Frequency Datasets (Forward-Fill):**
5. `{ASSET}USDT_fundingRate.csv` → `funding_rate` (8-hour, forward-filled)
6. `fear_greed_index.csv` → `fear_greed_value`, `fear_greed_classification` (daily, forward-filled)

**Text Data (Aggregated by Hour):**
7. `reddit_posts.csv` → `text_content` (concatenated with `[SEP]`, filled with placeholder)

**Base Index:**
8. `{ASSET}USDT_klines.csv` → OHLCV (master hourly index)

---

#### `calculate_continuous_target() → None`

Calculates continuous sentiment regression targets using **Volatility-Adjusted Tanh** formula:

$$y = \tanh\left(\frac{R}{k \cdot \sigma}\right) \times 100$$

Where:
- $R = \frac{\text{Close}_{\text{future}} - \text{Close}_{\text{current}}}{\text{Close}_{\text{current}}}$ (future return)
- $\sigma = \text{rolling\_std}(\text{hourly\_returns}, 168)$ (7-day volatility)
- $k = 1.5$ (scaling factor)

**Output:** `sentiment_score` column in [-100, +100]

**Edge Cases:**
- When $\sigma \approx 0$: Uses epsilon (1e-5) to avoid division by zero
- Rows at end where future data unavailable: Automatically dropped

---

#### `map_and_validate_images() → None`

Maps and validates candlestick chart images on disk:

**Image Path Format:**
```
data/processed/images/{asset_lower}/{unix_timestamp}.png
```

Example: `data/processed/images/btc/1672689600.png`

**Validation:**
- Uses `os.path.exists()` to verify each file on disk
- **CRITICAL**: Drops any rows where image file is missing (ensures 100% coverage)
- Logs: count found, count dropped

---

#### `assemble_final_dataset() → Tuple[Dataset, DataFrame]`

Assembles final 24-column dataset and converts to Hugging Face format.

**Final Columns (in order):**

| # | Column | Type | Description |
|----|--------|------|-------------|
| 1 | `timestamp` | datetime | UTC timestamp (hourly) |
| 2 | `asset` | str | "BTC" or "ETH" |
| 3-7 | `open`, `high`, `low`, `close`, `volume` | float | OHLCV |
| 8 | `funding_rate` | float | Perpetual funding rate |
| 9-12 | `open_interest_*` | float | Open interest OHLC |
| 13-14 | `long/short_liquidations` | float | Liquidated amounts |
| 15-17 | `long_short_ratio`, `long_percentage`, `short_percentage` | float | Trader sentiment |
| 18-19 | `news_volume`, `avg_sentiment_tone` | int/float | Macro news sentiment |
| 20-21 | `fear_greed_value`, `fear_greed_classification` | int/str | Fear/greed index |
| 22 | `text_content` | str | Hourly aggregated text |
| 23 | `sentiment_score` | float | Regression target [-100, +100] |
| 24 | `image_path` | str | Local image file path |

**Returns:**
- `(hf_dataset, df_final)`: HF Dataset (with images not yet cast) and pandas DataFrame

---

#### `cast_images_and_push_to_hub(...) → None`

Casts image column to `Image()` type and pushes to Hugging Face Hub.

```python
aligner.cast_images_and_push_to_hub(
    hf_dataset=hf_dataset,
    repo_id="khanh252004/multimodal_crypto_sentiment",
    private=False,
    dry_run=False
)
```

**Parameters:**
- `hf_dataset`: HuggingFace Dataset object
- `repo_id`: Target repository on Hub
- `private`: Whether to push as private dataset
- `dry_run`: If True, skips actual upload (useful for testing)

**Requires:** `HF_TOKEN` environment variable set

**Output:**
- Dataset accessible at: `https://huggingface.co/datasets/{repo_id}`
- Images embedded in dataset (can load with `dataset[0]["image_path"]`)

---

#### `run(...) → DataFrame`

Orchestrates complete pipeline end-to-end.

```python
df_final = aligner.run(
    push_to_hub=True,
    hub_repo_id="khanh252004/multimodal_crypto_sentiment",
    hub_private=False,
    hub_dry_run=False
)
```

**Phases Executed:**
1. Load all 8 data sources
2. Calculate continuous sentiment targets
3. Map and validate chart images
4. Assemble final 24-column dataset
5. Push to Hub (optional)

**Returns:** Final pandas DataFrame with all aligned data

## Troubleshooting

### Error: "HF_TOKEN not found in environment"
**Solution:** Set HF_TOKEN before running:
```bash
export HF_TOKEN='your_token'
python example_run_data_aligner.py --asset BTC
```

### Error: "Raw data directory not found"
**Solution:** Ensure `data/raw/` exists with all CSV files:
```
data/raw/
├── BTCUSDT_klines.csv
├── ETHUSDT_klines.csv
├── BTCUSDT_fundingRate.csv
├── ETHUSDT_fundingRate.csv
├── coinalyze_*.csv (3 files)
├── fear_greed_index.csv
├── gdelt_macro.csv.csv
└── reddit_posts.csv
```

### Error: "No images found for asset"
**Solution:** Generate chart images using ChartGenerator:
```python
from src.preprocessing.chart_generator import ChartGenerator

gen = ChartGenerator()
gen.generate_all_symbols()
```

### Large number of dropped rows due to missing images
**Solution:** Verify images were generated:
```bash
ls -la data/processed/images/btc/ | wc -l  # Count BTC images
```

## Architecture Notes

### Why Forward-Fill Only?

To prevent **future data leakage**, lower-frequency data (funding rates, fear/greed) are forward-filled, never backward-filled. This ensures:
- No future information leaks into past predictions
- Realistic model evaluation (model uses data available at prediction time)

### Why Drop Rows Without Images?

At deployment, both market data and chart images are required. Rows missing either are dropped:
- Ensures 100% coverage for multimodal models
- Prevents silent failures in production pipelines
- Simplifies data loading (no null handling needed)

### Continuous vs. Classification Targets

Regression targets (-100 to +100) provide:
- Richer signal for volatile markets
- Compatible with both regression and classification (can discretize)
- Better for uncertainty modeling
